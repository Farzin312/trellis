# Requirements Checklist: Product Readiness and Truthful Positioning

> Spec: 001

- [x] The problem is user-facing and contains no implementation prescription.
- [x] Every requirement has a stable ID and a verifiable outcome.
- [x] Success does not depend on “perfect,” universal, or unique claims.
- [x] The adopter journey, maintainer surface, generated project, and historical evidence are separated.
- [x] Required evidence and optional evidence have distinct outcomes.
- [x] Supported, compatible-with-wiring, optional, and out-of-scope capabilities are distinguishable.
- [x] Project identity and runtime support have a single contract.
- [x] Clean install, repeat install, partial failure, malformed input, mixed stacks, missing tools, paths, and corrupt ledgers are covered.
- [x] Every implementation task has a preceding runnable test task.
- [x] Every task names exact owned paths and requirement references.
- [x] External product compatibility is based on current official documentation and is explicitly drift-prone.
- [x] The plan deletes unsupported behavior before proposing replacement behavior.
- [x] Brownfield auto-merge is excluded because it would conflict with preservation requirements.
